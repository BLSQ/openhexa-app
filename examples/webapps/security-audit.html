Summary

Verify that bearer-token GraphQL queries work from a sandboxed context.

Get a signed token:
docker compose exec app python manage.py shell -c "from django.core.signing import Signer; from hexa.workspaces.models import WorkspaceMembership; m = WorkspaceMembership.objects.first(); print(Signer().sign_object(m.access_token))"

Simulate every known attack vector a malicious webapp could use. All should be blocked.

How the sandbox protects against each attack vector depending on context.

Attack Vector In Sandboxed Iframe Direct Navigation

Key: Static webapps are always loaded inside a sandboxed iframe without allow-same-origin. The "Direct Navigation" column shows what would happen if a user navigated directly to /webapps/<id>/ — this is mitigated by the UI never linking there directly. For full isolation against direct navigation, serve webapps from a separate subdomain.
